# Contributing to QuickTick

Thanks for your interest in contributing!

## Reporting Bugs

Open a [GitHub Issue](https://github.com/Uight/QuickTick/issues) for confirmed bugs or reproducible problems. Please include:

- A minimal reproduction or description of the problem
- Your OS, .NET version, and QuickTickLib version
- Observed vs. expected behavior

## Questions, Ideas, and Design Discussions

For questions, feature ideas, or anything that isn't a confirmed bug — please use [GitHub Discussions](https://github.com/Uight/QuickTick/discussions) instead of Issues. This keeps the issue tracker focused on actionable work.

## Security Vulnerabilities

Do not open a public issue for security problems. See [SECURITY.md](SECURITY.md) for how to report them privately.

## Pull Requests

1. Fork the repository and create a branch from `main`
2. Make your changes and ensure the existing tests still pass
3. Open a pull request with a clear description of what the change does and why

For anything non-trivial, it's worth opening a Discussion first to align on the approach before investing time in an implementation.
